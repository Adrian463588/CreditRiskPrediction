Workflow-CI Repository (Kriteria 3 - Advanced):
https://github.com/Adrian463588/Workflow-CI_SMSL

Struktur Repository:
Workflow-CI/
├── .github/
│   └── workflows/
│       └── ci_pipeline.yml        ← GitHub Actions CI (2 jobs: train + docker build)
└── MLProject/
    ├── MLProject                  ← MLflow Project manifest (entrypoint config)
    ├── conda.yaml                 ← Conda environment (reproducible dependencies)
    ├── Dockerfile                 ← Custom multi-stage Dockerfile (python:3.10-slim)
    ├── modelling.py               ← Training script entry point
    ├── docker_hub_reference.txt   ← Tautan Docker Hub image
    └── credit_risk_preprocessing/
        ├── credit_risk_train.csv  ← Dataset bersih untuk training
        └── credit_risk_test.csv   ← Dataset bersih untuk evaluasi

Docker Hub Image:
https://hub.docker.com/repository/docker/adrian090402/credit-risk-model/general

Pull Command:
docker pull adrian090402/credit-risk-model:latest

Catatan:
- CI Pipeline terpicu: push ke main + cron mingguan (Minggu 23:00 WIB)
- DevSecOps: semua credential via GitHub Secrets (tidak ada hardcoded values)
- Optimasi Docker: custom slim image ~500 MB vs default ~3 GB
